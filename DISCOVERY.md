# Discovery: organvm/organvm-ontologia

**Thesis:** `organvm-ontologia` is the working identity and observability spine of the live ORGANVM system — a stdlib-only, zero-dependency Python package that assigns permanent ULID-based UIDs to every entity in the 145-repo estate (organs, repos, modules, documents, sessions), preserves full temporal name histories so renames never break references, and ships a complete multi-sensor scanner (git, CI, filesystem, session, registry) feeding an inference engine that detects structural tensions (orphans, naming conflicts, overcoupling) governed by declarative evolution policies that trigger rename, merge, deprecate, and promote actions automatically. It is already in active production use — the live entity UIDs and system variables surfaced throughout `CLAUDE.md` are served by this package — making it the single strongest candidate for promotion as the shared substrate that all ORGANVM services depend on for entity resolution, event replay, and structural self-monitoring. The best concrete first task is implementing the empty `src/ontologia/cli/__init__.py` with `resolve`, `search`, `history`, and `scan` subcommands to expose the full registry capability as a standalone `ontologia` command accessible from any repo in the estate.
